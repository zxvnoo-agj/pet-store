# Feature Specification: 首页场景快捷卡（Delta Spec）

**Feature Branch**: `007-production-launch-prep`  
**Created**: 2026-05-29  
**Status**: Draft  
**Input**: User description: "当前的首页中有一个按钮为问问AI助手，点击后会跳转到AI助手tab，但这在小程序上不支持，因此现在将该按钮替换为场景快捷卡。把「问问 AI 助手」这块改成一个不跳转、直接服务首页购买决策的场景推荐模块。删除橙色 AI 横幅，把这块改成浅色功能区。卡片做成横向滑动。点击后直接筛选当前首页商品列表，或者进入分类/搜索结果页。"

## 变更背景

当前首页存在一个橙色的「问问 AI 助手」横幅卡片（位于宠物类型选择器下方、商品推荐列表上方）。该卡片点击后意图跳转到 AI 助手 Tab 页面，但在微信小程序环境下，`Taro.switchTab` 或 `Taro.navigateTo` 到 Tab 页存在体验问题或不被支持。为避免用户困惑并提升首页购物转化效率，现决定：

1. **移除**该 AI 助手横幅；
2. **替换**为一个横向滑动的「场景快捷推荐」模块；
3. 用户点击场景卡片后，**直接在当前首页原地过滤商品列表**，不涉及 AI 页面跳转。

---

## Clarifications

### Session 2026-05-29

- Q: 场景卡片点击后的跳转方式 → A: 原地过滤首页列表。点击场景卡后，当前首页商品列表立即按关键词过滤，场景卡区域高亮显示选中状态，支持一键清除筛选回到全部商品。
- Q: 场景卡片数量 → A: 6 个场景卡片（美毛亮毛、肠胃敏感、幼猫成长、囤货专区、挑食改善、换季护理）。
- Q: 其他宠物类型的场景配置 → A: v1 复用猫咪场景模板并替换关键词。非猫咪宠物类型（狗狗、鸟类、水族、爬宠、小宠）使用相同的 6 个场景结构，但将标题和关键词中的宠物类型名称做对应替换（如「幼猫成长」→「幼犬成长」）。v2 迭代再为各类型定制专属场景。
- Q: 清除筛选的交互方式 → A: 在场景区域标题行右侧显示「清除」按钮，仅当用户已选中某个场景时可见。点击后取消当前场景筛选，恢复展示全部商品列表，场景卡片恢复未选中状态。
- Q: 场景筛选的商品匹配逻辑 → A: 触发搜索 API。点击场景卡片后，系统调用搜索接口（如 `/v1/spus/search?keywords=...`）获取匹配商品，在当前首页展示搜索结果，需处理加载状态和空状态。不再仅在前端已加载的推荐列表中过滤。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 通过场景卡片快速找到目标商品 (Priority: P1)

用户在首页浏览时，看到根据自己当前选中的宠物类型（如猫咪）展示的场景快捷卡（如「美毛亮毛」「肠胃敏感」等）。用户点击感兴趣的场景卡片后，页面直接展示该场景相关的商品列表，无需进入 AI 对话页面。

**Why this priority**: 这是本次变更的核心价值——替代原 AI 助手入口，提供更直接、更稳定的购物路径，避免因小程序 Tab 跳转限制导致的体验中断。

**Independent Test**: 单独验证首页场景卡片模块：选择宠物类型 → 看到对应场景卡 → 点击卡片 → 页面展示过滤后的商品列表或跳转至搜索结果页。

**Acceptance Scenarios**:

1. **Given** 用户位于首页且已选中某宠物类型（如猫咪），**When** 页面加载完成，**Then** 在宠物类型选择器下方、商品推荐列表上方显示横向可滑动的场景快捷卡区域。
2. **Given** 用户看到场景快捷卡区域，**When** 用户左右滑动，**Then** 可以浏览所有场景卡片（每张卡片展示 emoji/图标、主标题、副标题关键词）。
3. **Given** 用户点击某张场景卡片（如「美毛亮毛」），**When** 点击生效，**Then** 系统调用搜索 API 获取匹配商品，并在当前首页原地展示搜索结果（展示 loading 状态后替换商品推荐区域）。若搜索结果为空，则展示空状态提示并提供「查看更多商品」入口。
4. **Given** 用户已点击某场景卡片并看到过滤结果，**When** 用户执行刷新或重新进入首页，**Then** 场景卡片区域恢复默认状态（未选中任何场景）。

---

### User Story 2 - 首页视觉与信息层级优化 (Priority: P2)

用户进入首页后，看到更简洁、清爽的界面布局。原高饱和度的橙色 AI 横幅已被移除，取而代之的是浅色背景的功能区。商品卡片的信息层级更清晰，价格、标题、评分等元素排布合理，不互相抢视线。

**Why this priority**: 提升首页整体视觉品质和可用性，减少视觉噪音，让用户更聚焦于商品发现和购买决策。

**Independent Test**: 单独验证首页视觉改版：进入首页 → 确认无橙色 AI 横幅 → 确认场景区域为浅色背景 → 确认商品卡片排版优化。

**Acceptance Scenarios**:

1. **Given** 用户进入首页，**When** 页面渲染完成，**Then** 不再显示原橙色渐变 AI 助手横幅（含「问问 AI 助手」文字和箭头图标）。
2. **Given** 用户查看场景快捷卡区域，**When** 页面渲染完成，**Then** 该区域背景色为浅橙/米色（如 `#FFF4EA`），而非高饱和橙色渐变。
3. **Given** 用户浏览商品推荐列表，**When** 查看单个商品卡片，**Then** 商品标题加粗但字号适中，品类/适用说明为浅灰色小字，价格位于右下角并以橙色突出显示，评分和评论数弱化显示。

---

### User Story 3 - 不同宠物类型展示对应场景 (Priority: P3)

用户切换首页顶部的宠物类型（如从猫咪切换到狗狗）后，场景快捷卡区域的内容随之更新，展示适合当前宠物类型的场景推荐。

**Why this priority**: 保证场景推荐的精准性和相关性，提升不同宠物主人的使用体验。

**Independent Test**: 单独验证宠物类型切换后的场景联动：在首页切换宠物类型 → 观察场景卡片内容是否变化。

**Acceptance Scenarios**:

1. **Given** 用户当前选中「猫咪」，**When** 用户切换选中「狗狗」，**Then** 场景快捷卡区域更新为狗狗相关的场景推荐（如狗狗美毛、幼犬成长等）。
2. **Given** 用户选中某种宠物类型，**When** 该宠物类型暂无预设场景配置，**Then** 场景快捷卡区域可以隐藏或展示通用默认场景（如「热门推荐」「日常用品」）。

---

### Edge Cases

- **无匹配商品场景**：用户点击某场景卡片后，当前商品列表中没有任何商品匹配该场景的关键词。此时应展示空状态提示（如「暂无相关商品，试试其他场景？」），并允许用户一键清除筛选回到全部商品。
- **网络异常导致商品加载失败**：若首页商品推荐列表加载失败，场景卡片仍应正常展示，但点击后需提示用户网络异常或展示缓存数据。
- **快速切换宠物类型**：用户快速连续切换宠物类型时，场景卡片的切换应平滑，不应出现闪烁或内容错位。
- **小程序胶囊按钮遮挡**：场景卡片横向滑动区域需确保在各类小程序（包括有顶部胶囊按钮的版本）中不被遮挡，可正常滑动和点击。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统**必须**从首页移除原橙色渐变 AI 助手横幅卡片（包含「问问 AI 助手」文字、AI 图标、箭头指示器）。
- **FR-002**: 系统**必须**在宠物类型选择器下方、商品推荐列表上方新增「场景快捷推荐」模块。
- **FR-003**: 场景快捷推荐模块**必须**支持横向滑动，每张卡片宽度约 150-170px，高度约 72-88px，圆角 16px。
- **FR-004**: 每张场景卡片**必须**包含：左侧 emoji/图标、右侧两行文字（主标题如「美毛亮毛」、副标题如「鱼油 / 美毛粮」）。
- **FR-005**: 场景快捷推荐模块的背景色**必须**使用浅色方案（如 `#FFF4EA` 浅橙色/米色），不得使用高饱和橙色渐变。
- **FR-006**: 用户点击场景卡片后，系统**必须**在当前首页原地展示搜索结果（不跳转新页面）。通过调用搜索 API（携带场景对应的关键词）获取匹配商品列表，替换当前首页的商品推荐区域。需处理加载状态（展示 skeleton/loading）和空状态（无匹配商品时展示空状态提示）。场景卡区域高亮显示当前选中的场景，同时提供「清除筛选」按钮允许用户一键恢复全部商品推荐列表。
- **FR-007**: 系统**必须**根据当前选中的宠物类型动态切换场景卡片的内容。v1 版本中，非猫咪的宠物类型（狗狗、鸟类、水族、爬宠、小宠）复用猫咪的 6 个场景模板，但将关键词中的「猫」替换为对应宠物类型名称（如「幼猫成长」→「幼犬成长」）。v2 迭代中为各宠物类型定制专属场景配置。
- **FR-008**: 首页顶部问候语**必须**从「下午好 👋 今天想给{petName}看点什么？」调整为更自然的表达，如「下午好，今天给{petName}挑点什么？」，其中宠物名称保留橙色强调。
- **FR-009**: 商品卡片（SpuCard）的视觉层级**必须**优化：标题加粗但字号适中，品类/适用说明使用浅灰色小字，价格放置于右下角并以橙色突出，评分和评论数弱化显示。
- **FR-011**: 当用户已选中某场景卡片时，系统**必须**在场景快捷推荐区域的标题行右侧显示「清除」按钮。点击该按钮后，取消当前场景筛选，恢复展示全部商品列表，所有场景卡片恢复未选中状态。
- **FR-010**: 系统**必须**展示 6 个场景卡片（猫咪类型下）。横向滑动区域需完整展示 6 张卡片，用户可左右滑动浏览全部场景。

### Key Entities *(include if feature involves data)*

- **Scenario Card（场景卡片）**: 代表一个宠物购买场景推荐，包含以下属性：
  - `id`: 唯一标识
  - `petType`: 适用的宠物类型（如 cat, dog）
  - `icon`: 图标/emoji
  - `title`: 主标题（如「美毛亮毛」）
  - `subtitle`: 副标题/关键词（如「鱼油、卵磷脂、美毛粮」）
  - `filterKeywords`: 用于筛选商品的关键词列表
  - `targetPageTitle`: 跳转后的页面标题（如「美毛亮毛推荐」）
- **Pet Type（宠物类型）**: 当前首页选中的宠物类型，决定展示哪些场景卡片。当前系统已支持 cat, dog, bird, fish, reptile, small_pet。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 用户从首页进入特定场景商品列表的操作步数不超过 2 步（点击场景卡 → 看到商品）。
- **SC-002**: 首页首屏加载时间（包含场景卡片渲染）在 3G 网络环境下不超过 3 秒。
- **SC-003**: 用户能在 5 秒内从场景卡片区域识别出符合自己需求的场景并点击。
- **SC-004**: 移除 AI 横幅后，首页首屏的视觉噪点评分（可通过用户访谈或 A/B 测试衡量）提升至少 20%。
- **SC-005**: 场景卡片点击率（点击场景卡的用户数 / 看到场景卡的用户数）在上线后 2 周内达到 15% 以上。

---

## Assumptions

- 后端已提供或将在本次迭代中提供按关键词搜索商品的 API 接口（如 `/v1/spus/search?keywords=...`），支持首页场景卡片的搜索过滤需求。
- 场景卡片的配置数据（标题、关键词、图标等）为静态配置，v1 版本无需后端动态管理，可直接硬编码在前端或使用本地 JSON 配置。
- 商品卡片组件 `SpuCard` 的样式调整不会影响其他使用到该组件的页面（如搜索结果页、收藏页等），或影响在可接受范围内。
- 微信小程序环境对横向滚动组件（`scroll-view` 或 CSS `overflow-x`）的支持正常，不会出现明显的兼容性问题。
- 当前已支持的宠物类型中，除猫咪外，其他类型的场景配置可以在后续迭代中补充，v1 版本允许这些类型暂时展示通用默认场景或不展示场景模块。
