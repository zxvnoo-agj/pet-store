# Feature Specification: 宠物档案与AI能力增强

**Feature Branch**: `006-pet-profiles-ai`  
**Created**: 2026-05-24  
**Status**: Draft  
**Input**: User description: "本特性旨在强化用户信息和AI能力。首先用户注册时可以添加自己的宠物，包括选择宠物种类，品种，其他信息。在小程序我的页面增加宠物管理页。若用户进入首页且有自己的宠物，则在在首页替换当前的宠物选择tab栏，展示用户的宠物卡片。下面的推荐以用户的宠物类型推荐spu。宠物卡片留有选择其他按钮，用户如果不是为当前宠物选择物品，点击可以选择 宠物种类。用户在与AI交互时，如果用户绑定了宠物信息，宠物信息将自动注入上下文，用户如果和AI交谈，会有该部分信息。此外，当前AI助手界面输入框上面的推荐问题现在是写死的，在有用户的宠物信息后可以由AI生成推荐问题展示。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 注册时添加宠物信息 (Priority: P1)

用户首次登录注册时，系统引导用户添加自己的宠物档案，包括选择宠物种类（猫/狗/鸟/鱼等）、品种及其他信息（昵称、年龄、体重等）。注册完成后，宠物信息与用户账号绑定。

**Why this priority**: 宠物档案是整个特性的数据基础。首页个性化推荐、AI上下文注入、AI推荐问题生成都依赖宠物信息。没有宠物信息，后续功能无法生效。

**Independent Test**: 注册流程中可独立验证宠物信息表单的填写和提交，确认数据正确存储并与用户关联。

**Acceptance Scenarios**:

1. **Given** 用户首次微信登录，**When** 登录成功后，**Then** 系统展示"添加你的宠物"引导页面，用户可选择宠物种类（猫/狗/鸟/鱼等）
2. **Given** 用户在添加宠物页面选择了"猫"，**When** 选择品种（如"英短"、"美短"），填写昵称、年龄、体重等可选信息后提交，**Then** 宠物信息保存成功，跳转至首页
3. **Given** 用户跳过添加宠物步骤，**When** 直接进入首页，**Then** 首页展示默认宠物选择Tab栏（与当前行为一致）
4. **Given** 用户已绑定宠物，**When** 再次登录，**Then** 不重复展示添加宠物引导

---

### User Story 2 - 我的页面宠物管理 (Priority: P1)

用户可以在"我的"页面查看和管理自己的宠物档案，支持添加新宠物、编辑已有宠物信息、删除宠物。

**Why this priority**: 宠物管理是用户维护宠物档案的唯一入口，与注册引导形成完整闭环。用户需要能随时修改宠物信息。

**Independent Test**: 在"我的"页面操作宠物管理功能，可独立验证增删改查全流程。

**Acceptance Scenarios**:

1. **Given** 用户已登录且绑定宠物，**When** 进入"我的"页面，**Then** 在功能列表中展示"宠物管理"入口，显示当前宠物数量
2. **Given** 用户进入宠物管理页，**When** 页面加载，**Then** 展示已绑定的宠物列表（卡片形式，含种类图标、品种、昵称）
3. **Given** 用户在宠物管理页，**When** 点击"添加宠物"，**Then** 弹出宠物信息表单（种类、品种、昵称、年龄、体重等），提交后新增宠物
4. **Given** 用户有已绑定的宠物，**When** 点击宠物卡片的编辑按钮，**Then** 可修改该宠物的全部信息并保存
5. **Given** 用户有已绑定的宠物，**When** 点击删除，确认后，**Then** 该宠物档案被移除

---

### User Story 3 - 首页宠物卡片个性化推荐 (Priority: P2)

当用户已绑定宠物时，首页顶部用宠物卡片替换当前的默认宠物选择Tab栏。展示用户的每只宠物卡片（含种类图标、品种、昵称），默认选中用户上次选择的宠物或第一只宠物，推荐内容基于选中宠物的种类进行个性化推荐。宠物卡片末尾保留"选择其他"入口，允许用户临时选择非绑定品种浏览商品。

**Why this priority**: 首页个性化推荐直接提升用户体验，但依赖P1的宠物档案建立。对于未绑定宠物的用户，体验与当前一致不降级。

**Independent Test**: 分别以有宠物和无宠物的用户身份访问首页，验证两种模式下的展示和推荐逻辑正确。

**Acceptance Scenarios**:

1. **Given** 用户已登录且绑定2只猫，**When** 进入首页，**Then** 顶部展示2只猫的宠物卡片（含昵称、品种、种类图标），替代默认宠物选择Tab栏
2. **Given** 用户选中某只布偶猫卡片，**When** 页面加载推荐区，**Then** 推荐内容基于"猫"类型筛选SPU，展示标题如"给【昵称】的推荐"
3. **Given** 用户有绑定宠物，**When** 点击宠物卡片末端的"选择其他"按钮，**Then** 弹出宠物种类选择面板（猫/狗/鸟/鱼），用户可选择任意种类浏览对应推荐
4. **Given** 用户未登录或无绑定宠物，**When** 访问首页，**Then** 展示当前默认的宠物选择Tab栏（猫咪/狗狗/鸟类/水族），体验不变

---

### User Story 4 - AI对话宠物上下文注入 (Priority: P2)

用户与AI助手对话时，系统自动将用户绑定的宠物信息注入上下文，使AI能够基于用户宠物的种类、品种、年龄等给出个性化建议。

**Why this priority**: 宠物上下文注入让AI推荐从"通用推荐"升级为"精准推荐"，是小程序AI助手核心差异化能力的增强。

**Independent Test**: 绑定宠物的用户与未绑定宠物的用户分别发起AI对话，验证AI上下文和回复的差异。

**Acceptance Scenarios**:

1. **Given** 用户已登录且绑定1只3个月大的英短猫（昵称"团子"），**When** 用户向AI发送"推荐猫粮"，**Then** AI系统提示中自动包含用户的宠物信息（种类、品种、年龄），AI回复可针对幼猫阶段推荐
2. **Given** 用户有绑定宠物，**When** 用户发送"我家猫最近不爱吃饭怎么办"，**Then** AI回复中结合宠物的品种和年龄信息给出针对性建议
3. **Given** 用户未登录或无绑定宠物，**When** 发起AI对话，**Then** AI使用通用系统提示（与当前行为一致）
4. **Given** 用户有多只猫，**When** 发起AI对话，**Then** AI上下文中包含所有宠物信息，AI可在回复中针对不同宠物分别建议

---

### User Story 5 - AI动态推荐问题 (Priority: P3)

AI助手界面输入框上方的推荐问题，从当前固定的4条静态问题改为基于用户宠物信息动态生成。有绑宠用户看到个性化推荐问题（如"幼猫猫粮选什么品牌？"），未绑宠用户看到通用推荐问题。

**Why this priority**: 动态推荐问题能提升用户提问效率和AI助手的使用率，但不影响核心对话功能。P3优先级可在前两个需求完成后实现。

**Independent Test**: 对比有宠物和无宠物的用户访问AI助手时推荐问题的差异，验证问题与宠物信息的相关性。

**Acceptance Scenarios**:

1. **Given** 用户已绑定一只3个月英短猫，**When** 进入AI助手对话页，**Then** 推荐问题区域展示与"幼猫""英短"相关的个性化问题（如"3个月英短适合什么猫粮？"、"英短幼猫每天喂多少？"）
2. **Given** 用户未绑定宠物，**When** 进入AI助手对话页，**Then** 展示通用推荐问题（如"3个月幼猫推荐什么猫粮？"、"皇家和渴望哪个好？"）
3. **Given** 用户有多只不同种类的宠物，**When** 进入AI助手对话页，**Then** 推荐问题可覆盖不同宠物的需求
4. **Given** 用户点击某个动态推荐问题，**When** 问题发送至AI，**Then** AI回复中结合宠物信息给出个性化回答

---

### Edge Cases

- **无宠物用户**: 首页和AI助手保持当前行为不变，不展示宠物卡片、不使用宠物上下文，推荐问题保持通用
- **多宠物用户**: 最多支持[N]只宠物（默认5只），超出时提示"最多添加5只宠物"
- **宠物信息不完整**: 用户仅填写了种类未填写品种/年龄时，AI上下文仅注入已有信息，不编造缺失内容
- **品种选择范围**: 品种列表按种类联动（先选种类再选品种），品种数据应覆盖主流宠物品种
- **宠物删除影响**: 删除宠物后，清除该宠物在首页的卡片展示，AI上下文中移除该宠物信息
- **宠物编辑同步**: 修改宠物信息后，首页宠物卡片和AI上下文实时反映最新数据
- **动态推荐问题生成失败**: 当AI生成推荐问题接口异常时，降级展示默认通用推荐问题
- **AI上下文长度控制**: 多宠物信息注入时，需控制上下文长度避免超限，超过限制时截断附加描述
- **匿名用户**: 未登录用户无法添加宠物，所有宠物相关功能降级为通用模式
- **重复宠物**: 用户尝试添加同种类且同昵称的宠物时，系统提示"已存在同名宠物"，阻止重复创建

## Clarifications

### Session 2026-05-24

- Q: 宠物档案的唯一性约束规则？ → A: 同一用户（user, species, nickname）组合不可重复。即同一用户不能有两只同种类且同昵称的宠物。
- Q: 多宠物时首页默认选中哪只宠物？ → A: 记住用户上次选中的宠物（跨会话持久化），若无历史记录则默认选中第一只。
- Q: AI推荐问题生成失败时用户是否感知？ → A: 静默降级，展示固定的通用推荐问题，不提示生成失败。用户无感知。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统必须支持用户在注册登录后添加宠物档案，包含种类（必选）、品种（可选）、昵称（可选）、年龄（可选）、体重（可选）、备注（可选，自由文本）
- **FR-002**: 系统必须支持宠物种类的层级联动选择：先选择种类，再选择对应品种。品种数据通过数据库表维护，管理后台可增删改品种
- **FR-003**: 系统必须在"我的"页面提供宠物管理入口，展示已绑定宠物数量
- **FR-004**: 系统必须支持宠物档案的增删改查操作（添加、编辑、删除、查看宠物列表）。同一用户不允许添加种类和昵称完全相同的宠物（唯一约束：user + species + nickname）
- **FR-005**: 系统必须限制每位用户最多绑定5只宠物
- **FR-006**: 首页必须根据用户是否有绑定宠物切换两种展示模式：有宠物时展示宠物卡片，无宠物时展示默认宠物选择Tab栏
- **FR-007**: 首页宠物卡片必须展示宠物的种类图标、品种、昵称，并基于选中的宠物种类进行SPU推荐
- **FR-008**: 宠物卡片列表末尾必须保留"选择其他"入口，点击后可临时选择非绑定宠物种类浏览推荐
- **FR-009**: AI对话时，系统必须自动将用户绑定的宠物信息注入系统提示上下文
- **FR-010**: 宠物信息注入AI上下文必须包含：种类、品种、昵称、年龄、体重、备注（仅注入用户已填写的信息）
- **FR-011**: 无宠物用户与AI对话时，系统提示保持当前通用模式，不注入宠物信息
- **FR-012**: AI助手推荐问题必须支持两种模式：有宠物用户展示AI动态生成的个性化问题，无宠物用户展示默认通用问题
- **FR-013**: AI动态推荐问题采用混合策略：优先使用缓存（有效期24小时），缓存失效时实时生成；生成失败时静默降级展示固定的通用推荐问题，不提示用户
- **FR-014**: 首页和AI助手页面必须实时反映宠物信息的最新变更

### Key Entities

- **Pet Profile**: 用户宠物档案，关联到用户。包含种类（cat/dog/bird/fish等）、品种（如英短、布偶）、昵称、年龄、体重、备注等信息。与用户为多对一关系。唯一约束：同一用户的 (species, nickname) 组合不可重复
- **Pet Breed**: 宠物品种数据，按种类分类存储，支持管理后台维护。为品种选择提供标准化数据源
- **User (扩展)**: 用户实体扩展宠物档案关联
- **Chat Session (扩展)**: AI对话会话需感知用户宠物上下文，系统提示中动态注入宠物信息

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 用户在注册流程中完成宠物信息添加的平均时间不超过1分钟
- **SC-002**: 宠物管理页面的增删改查操作响应时间不超过2秒
- **SC-003**: 有宠用户的首页在3秒内完成个性化推荐加载
- **SC-004**: 有宠用户使用AI助手后，首次推荐的相关性评分提升30%（相比无宠物通用提示）
- **SC-005**: 动态推荐问题的点击率高于固定推荐问题20%以上
- **SC-006**: 无宠用户的首页和AI助手体验相对于变更前无功能降级
- **SC-007**: 宠物信息注入AI上下文的延迟不超过500ms（对AI对话响应时间无感知影响）
- **SC-008**: 动态推荐问题生成在2秒内完成（从进入AI助手页面到问题展示）

## Assumptions

- 宠物种类与现有分类体系的pet_type字段对齐（cat/dog/bird/fish等）
- 品种数据通过数据库表管理，管理后台可增删改品种（已确认：Q1 选项B）
- AI动态推荐问题采用混合策略：优先使用缓存（24小时有效期），缓存失效时实时生成（已确认：Q2 选项C）
- 宠物档案扩展支持核心字段+备注字段（已确认：Q3 选项C）
- 现有首页的默认宠物选择Tab栏逻辑保留，仅在有宠物时替换展示
- 用户未登录时所有宠物相关功能隐藏或降级
- 宠物信息存储采用独立存储结构（非Users表的单一字段），以支持增删改查操作
